25 /// @brief PosgreSQL client component
26 ///
27 /// Provides access to a PostgreSQL cluster.
28 ///
29 /// ## Dynamic options:
30 /// * @ref POSTGRES_DEFAULT_COMMAND_CONTROL
31 /// * @ref POSTGRES_HANDLERS_COMMAND_CONTROL
32 /// * @ref POSTGRES_QUERIES_COMMAND_CONTROL
33 /// * @ref POSTGRES_CONNECTION_POOL_SETTINGS
34 /// * @ref POSTGRES_STATEMENT_METRICS_SETTINGS
35 ///
36 /// ## Static configuration example:
37 ///
38 /// ```
39 /// # yaml
40 /// postgres-taxi:
41 /// dbalias: taxi
42 /// blocking_task_processor: task-processor-name
43 /// max_replication_lag: 60s
44 /// min_pool_size: 4
45 /// max_pool_size: 15
46 /// max_queue_size: 200
47 /// max_statement_metrics: 50
48 /// ```
49 /// You must specify either `dbalias` or `conn_info`.
50 /// If the component is configured with an alias, it will lookup connection data
51 /// in secdist.json
52 ///
53 /// You must specify `blocking_task_processor` as well.
54 ///
55 /// `max_replication_lag` can be used to tune replication lag limit for replicas.
56 /// Once the replica lag exceeds this value it will be automatically disabled.
57 /// Note, however, that client-size lag detection is not precise in nature
58 /// and can only provide the precision of couple seconds.
59 ///
60 /// ## Secdist format
61 ///
62 /// A PosgreSQL alias in secdist is described as a JSON array of objects
63 /// containing a single cluster description. There are two formats of describing
64 /// a cluster, the first one assigns predefined roles to DSNs, the second one
65 /// is just a list of DSNs and the Postgres component takes care of discovering
66 /// the cluster's topology itself.
67 ///
68 /// ### Predefined roles
69 ///
70 /// In predefined roles format the component requires single-host connection
71 /// strings.
72 ///
73 /// ```json
74 /// {
75 /// "shard_number" : 0,
76 /// "master": "host=localhost dbname=example",
77 /// "sync_slave": "host=localhost dbname=example",
78 /// "slaves": [
79 /// "host=localhost dbname=example"
80 /// ]
81 /// }
82 /// ```
83 ///
84 /// The predefined roles format is deprecated and the support will be removed as
85 /// soon as all ¬µservices migrate to the automatical discovery format.
86 ///
87 /// ### Automatic discovery
88 ///
89 /// In automatic discovery format the connection strings are any valid
90 /// PostgreSQL connection strings including multi-host ones with the exception
91 /// of `target_session_attrs` which will be ignored.
92 ///
93 /// ```json
94 /// {
95 /// "shard_number" : 0,
96 /// "hosts": [
97 /// "host=host1,host2,host3 dbname=example",
98 /// "postgresql://host1:5432,host2:6432,host3:12000/example"
99 /// ]
100 /// }
101 /// ```
102 ///
103 /// The `shard_number` parameter is required in both formats and must match the
104 /// index of cluster description object in the alias array.
105 ///
106 /// Please see [PostgreSQL documentation](https://www.postgresql.org/docs/12/libpq-connect.html#LIBPQ-CONNSTRING)
107 /// on connection strings.
108 ///
109 /// ## Static options:
110 /// Name | Description | Default value
111 /// ----------------------- | --------------------------------------------------------- | -------------
112 /// dbalias | name of the database in secdist config (if available) | --
113 /// dbconnection | connection DSN string (used if no dbalias specified) | --
114 /// blocking_task_processor | name of task processor for background blocking operations | --
115 /// max_replication_lag | replication lag limit for usable slaves | 60s
116 /// min_pool_size | number of connections created initially | 4
117 /// max_pool_size | limit of connections count | 15
118 /// sync-start | perform initial connections synchronously | false
119 /// dns_resolver | server hostname resolver type (getaddrinfo or async) | 'getaddrinfo'
120 /// persistent-prepared-statements | cache prepared statements or not | true
121 /// user-types-enabled | disabling will disallow use of user-defined types | true
122 /// ignore_unused_query_params| disable check for not-NULL query params that are not used in query| false
123 /// monitoring-dbalias | name of the database for monitorings | calculated from dbalias or dbconnection options
124 /// max_prepared_cache_size | prepared statements cache size limit | 5000
125 /// max_statement_metrics | limit of exported metrics for named statements | 0
126 /// min_pool_size | number of connections created initially | 4
127 /// max_pool_size | maximum number of created connections | 15
128 /// max_queue_size | maximum number of clients waiting for a connection | 200
